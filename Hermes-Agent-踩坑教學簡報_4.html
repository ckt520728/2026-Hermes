引言 · INTRODUCTION

Harness ＝ AI 的作業系統

2026 的範式轉移：模型被商品化，勝負移到「包在模型外面那層」。一句話——Agent ＝ Model ＋ Harness。

Agent跑在上面的具體應用，例如 Hermes
App
Harness工具驅動 · 記憶 · 排程 · 安全 · 容錯
OS
Context Window有限的工作記憶
RAM
Model原始推理力（凍結的計算器）
CPU
這個「OS」在管什麼？
工具調度記憶持久化安全 / 人類審核 多步編排子代理容錯降級
模型負責「推理」，harness 負責「讓推理變成可靠、可重複的行動」。
模型再聰明，沒有好的 OS，也跑不出可靠的 agent。

＊ 概念來源：OpenAI Codex 團隊「harness engineering」、Philipp Schmid 的電腦類比（2026）。

安裝層 · 環境關卡（坑 1–3）

別跟 pip 糾纏，一行腳本全包

坑 1
pip 不在 PATH：cmd 打 pip install 噴「不是內部或外部命令」。Anaconda 的 Scripts 沒進 cmd 的 PATH。
坑 2
Python 3.9 太舊：Hermes 需要 3.11+，純 pip 裝會失敗。
坑 3
cmd 不認 &&：那是 PowerShell 7／bash 語法。→ 改用 PowerShell。
PowerShell（不要用 cmd）
# 一行搞定：自動裝齊 Python 3.11 + Node.js + ripgrep + ffmpeg + uv
# 不會動到你既有的 Anaconda 環境
PS> irm https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.ps1 | iex

＊ 裝完重開一個 PowerShell，PATH 變更才會生效。

模型層 · 關鍵 ★（坑 10–13）

四層模型設定，改一層不夠

預設用付費的 Claude Opus 4.6 → 餘額不足噴 404。改 Gemini 要層層落實：

10Main Model
對話主模型。改成免費的 google/gemini-3-flash-preview。
11Auxiliary Tasks ★
11 項輔助任務各自獨立，預設全 = Claude。逐項 override 成 Gemini，狀態列要顯示 11 overrides · 0 auto。
12Profile 欄位
名稱與模型是分開兩欄。google/... 含 / 填到名稱欄會被 reject。
13CRON 快照
舊任務把模型寫死在建立當下。改設定後要砍掉重建才會吃新值。

🚨 千萬別按 RESET ALL TO AUTO，會打回原狀、前功盡棄。

模型層 · 三個觀念修正（坑 8 · 9 · 14）

三個「以為是這樣，其實不是」

坑 8
Kanban ≠ 分類看板。它是 AI 任務佇列（待分類→待辦→已排程→完成），每張卡是一個任務。
定時抓資訊 → 用 CRON；丟具體任務 → 用 Kanban。
坑 9
CRON 不能寫 0900。要 5 個欄位用空白隔開（分 時 日 月 週）。每天 9:00 = 0 9 * * *。不熟就上 crontab.guru。
坑 14
@BotFather 不是你的 bot。它只是「註冊工具」。要去你自己建的那個 bot 的對話框按 START，才會收到 Hermes 回應。

工作流哲學 · 最關鍵 ★★（坑 15 · 17）

觸發點，決定回報點

CRON 跑成功、Dashboard 綠燈，但 Telegram 收不到（Chat not found）。差別只在你從哪裡觸發。

✕ 反模式

  • 建 CRON 用預設 deliver: telegram
  • 早上自動跑 → 結果消失在 Home Channel
  • Dashboard 按 ▶︎ Run now → 一樣消失
  • 以為系統壞了

✓ 正確

  • 建 CRON 時 deliver: origin
  • 不管從哪觸發，都回到觸發點
  • 想立刻重跑 → 在 Telegram 對 bot 說「跑一下」
  • 結果秒回到你眼前

＊ 對話觸發 = 執行 + 回報路徑自動設成當前對話，一氣呵成，最不會出錯。

提煉 · 六個關鍵教訓

如果只記住六件事

  • Windows 水土不服：文件多預設 Linux／macOS，自己翻譯 start→run、ui→dashboard、跳過 sudo／systemd。
  • 多層模型的隱形成本：Main／Auxiliary／Profile／任務快照，改一層不夠。
  • 對話導向 vs 視覺導向：Hermes 是前者，SaaS 思維會撞牆。
  • 「沒 web search」≠「不能上網」：指定 API endpoint 給它（真實），別叫它「搜尋」（編造）。
  • Agent 自己修錯的威力：一句「幫我修正傳送頻道」，它自己診斷、跑工具鏈、改設定——這就是 agentic AI 的價值。
  • PubMed E-utilities 是金礦：免費、無 key、官方、能抓完整 abstract，醫療研究自動化的 daily driver。

Apps Script 排程與 API Key 完美設定

解決最後一步的 PubMed 頻率限制 (Rate Limit Access)，以及一鍵自動定時早上 7 點排程。

01 解決 PubMed 頻率限制 (NCBI Key)
無 Key 限制為每秒 3 次。有 Key 提升至 10 次。
快捷解決方案：已在 daily-brief.gs 底部新增 setNcbiApiKeyHelper() 函式。
直接在代碼填入您的 Key 執行一次，金鑰即會儲存在雲端，安全且不需要留存程式碼中。
02 早上 7:00 排程一鍵自動設定
傳統手動點網頁觸發器費時又易錯。
快捷解決方案：直接執行 setupDailyTrigger() 函式，程式會自動清除重複舊觸發器，並在 Google 雲端建立每天早上 7:00 ~ 8:00 間運行的定時排程。
